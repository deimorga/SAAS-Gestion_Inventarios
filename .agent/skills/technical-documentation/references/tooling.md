# Guía de Herramientas de Documentación

Fuente: https://www.mintlify.com/blog/top-7-api-documentation-tools-of-2025

Usar este archivo al decidir expectativas de construcción/revisión para plataformas de docs.

## Puntos de Verificación para Selección de Herramientas

- Dependencia de stack existente: no forzar migración por ganancias menores.
- Profundidad de flujo de trabajo API: referencias generadas, soporte OpenAPI, capacidad de prueba.
- Modelo de colaboración: docs-como-código, flujo de revisión, versionamiento.
- Calidad en tiempo de ejecución: búsqueda, navegación y snippets de código listos para copiar.
- Preparación para IA: contenido estructurado, URLs estables, diseño amigable para máquinas pero legible por humanos.
- Preparación para humanos: complejidad de lectura, UX de lectura, profundidad de navegación, minimizar jerga.

## Aplicar en Modo Brownfield

- Priorizar compatibilidad con la plataforma actual.
- Usar componentes disponibles y convenciones de estilo antes de introducir nuevos patrones.
- Proponer migración solo cuando las restricciones actuales bloqueen resultados críticos.

## Aplicar en Modo Evergreen

- Favorecer plataformas y plantillas que hagan las actualizaciones rutinarias de baja fricción.
- Estandarizar plantillas de sección para reducir desviaciones.
- Capturar propiedad, cadencia de actualización y reglas de detección de contenido obsoleto.

## Implicaciones de Revisión

- Verificar que el contenido usa correctamente las primitivas de la plataforma (pestañas, callouts, bloques de endpoint).
- Señalar docs que son técnicamente correctas pero difíciles de escanear en la plataforma elegida.
- Recomendar mejoras específicas de plataforma solo cuando reduzcan la carga cognitiva.
